# Strummy — Unbreakable Core (P0/P1 must-pass scenarios)

_The contract from the 2026-05-10 grill. These scenarios must stay green. The
doc is the source of truth; `scripts/testing/check-unbreakable.cjs` walks it
and reports drift._

**Scope** (in priority order, accepted during the grill):

1. Teacher's lesson + student loop — the heartbeat.
2. Auth — the door.
3. Authorization, with shadow users + password creation as the spotlight.

Out of scope by explicit user direction:

- Student practice loop (C)
- Sync surface (D — Calendar / Spotify / Drive)
- AI features (E)

Status legend: ✅ covered (green test exists) · ⚠️ partial (some coverage,
gap noted in **Notes**) · ❌ missing (no test today).

## Roll-up

- **45 P0/P1 scenarios** across the three sections.
- **Today: 31 ✅ · 1 ⚠️ · 13 ❌ (68.9 % green).**
- The 13 ❌ are no longer "missing test, write it" — each one is blocked on a specific category boundary, called out in its row's note. Categories:
  - **NEEDS REAL POSTGRES** (3): `authz:fk-transfer-atomicity`, `authz:invite-email-auto-link-on-signup`, `auth:signup-atomic` — the unbreakable property lives in a SQL routine or trigger that no application-layer mock can exercise. Unblock by adding a Postgres test rig (e.g. `pgTAP` or a Supabase test branch) and writing the seeded-state tests under `supabase/tests/`.
  - **CODE CHANGE NEEDED** (3): `notes:auto-save-no-double-write`, `create-lesson:gcal-dedup`, `auth:mfa-bypass-impossible` — the test would lock behaviour the app does not yet have. Each row spells out the fix.
  - **NEW FEATURE** (1): `repertoire:sort-order-persists` — needs a `reorderRepertoireAction` that does not exist.
  - **SCHEMA MIGRATION** (1): `authz:invite-email-uniqueness` — replace the non-unique partial index with a UNIQUE one. Audit prod for duplicates first.
  - **PRODUCT DECISION** (2): `create-lesson:idempotent-double-click`, `create-lesson:past-date-policy` — pick a behaviour, _then_ test it. Both rows enumerate the options.
  - **PLAYWRIGHT** (3): `notes:no-optimistic-lie`, `create-lesson:visible-to-assigned-student`, `auth:signout-invalidates-session` — only browser-level tests can witness the property. Suggested file paths in each row.
- The 1 ⚠️ is `auth:password-reset-flow` — the application boundary (no enumeration in the response) is locked; the link-single-use + sign-in-after-reset paths require Playwright.

Run `npm run check:unbreakable` for a current status print-out. The script is report-only by design — it surfaces drift (a green row whose test file is missing) but does not block CI on remaining ❌. Add `--strict` once the ❌ count is materially lower to fail CI on any P0 gap.
