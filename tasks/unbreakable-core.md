## Roll-up

- **45 P0/P1 scenarios** across the three sections.

- Each scenario is locked by a Jest unit test, a Jest integration test, a Playwright spec, or a structural contract test against the migration SQL. Strength tier per row, ordered weakest → strongest:
  - **Structural** (3) — `authz:fk-transfer-atomicity`, `authz:invite-email-auto-link-on-signup`, `auth:signup-atomic`. The unbreakable property lives in plpgsql (`handle_new_user` trigger, `transfer_shadow_profile_references` function).   - **Helper / opt-in** (4) — `notes:auto-save-no-double-write`, `create-lesson:gcal-dedup`, `create-lesson:idempotent-double-click`, `auth:mfa-bypass-impossible`. The behaviour is shipped as a pure helper / wrapper (`lib/utils/dedup.ts`, `lib/utils/idempotency-cache.ts`, `lib/auth/withApiAuthMFA.ts`) plus a deployed demonstration route. Routes opt in by calling the helper before mutation; the wrapper is in production use at `/api/admin/aal-status`. Promote to "fully wired" by migrating each privileged route over.
  - **Drafted migration** (1) — `authz:invite-email-uniqueness`. App-level uniqueness is locked by `route-patch-invite.unit.test.ts`. The DB-level UNIQUE partial index migration is at `supabase/migrations/20260510130000_invite_email_unique.sql`, gated by the pre-flight duplicate-audit SELECT in the file header (run before applying).
  - **Playwright** (4) — `notes:no-optimistic-lie`, `create-lesson:visible-to-assigned-student`, `auth:signout-invalidates-session`, `auth:password-reset-flow`. Specs live under `tests/e2e/`; runnable with `npx playwright test` against a dev server. Two have `test.skip()` legs that need a test inbox or a stable seed pairing — see each row's note.
  - **Direct unit / integration** (33) — every other scenario is locked by a Jest unit or integration test against existing application code or the helper directly.
