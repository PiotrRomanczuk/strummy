## Journey 10: Spotify Match Review

**Priority**: P1
**Role**: Admin
**Pages**: `/dashboard/admin/spotify-matches`
**Existing coverage**: None

### Preconditions
- Logged in as admin
- At least one song has a pending Spotify match (confidence < 85%)

### Happy Path

#### Step 1 — Navigate to Spotify matches
1. Go to `/dashboard/admin/spotify-matches`
2. **Expect**: Page loads with a list/table of pending matches
3. **Expect**: Each match shows: song title, proposed Spotify track name, artist, confidence score, album art
4. **Expect**: "Approve" and "Reject" buttons per match

#### Step 2 — Review match details
1. Click on a match to expand or view details
2. **Expect**: Side-by-side comparison: your song (title, author) vs Spotify match (track, artist, album)
3. **Expect**: Confidence score visible (e.g., 72%)
4. **Expect**: Preview playback button (if Spotify preview URL available)

#### Step 3 — Approve a match
1. Click "Approve" on a match
2. **Expect**: Match status changes to "approved"
3. **Expect**: Song's `spotify_link_url` is updated with the approved Spotify URL
4. **Expect**: Match disappears from pending list (or moves to approved section)
5. Navigate to the song's detail page
6. **Expect**: Spotify link now visible on the song

#### Step 4 — Reject a match
1. Click "Reject" on a different match
2. **Expect**: Match status changes to "rejected"
3. **Expect**: Match disappears from pending list
4. **Expect**: Song's `spotify_link_url` remains empty

#### Step 5 — Trigger new sync
1. Click "Sync" / "Re-scan" button (if available)
2. **Expect**: Streaming progress shows scanning songs without Spotify links
3. **Expect**: New matches appear in the pending list after scan completes

### Edge Cases

#### E1 — No pending matches
1. If all matches are reviewed, navigate to the page
2. **Expect**: Empty state message ("No pending matches")

#### E2 — Auto-approved matches (>= 85% confidence)
1. Verify that high-confidence matches are auto-applied and don't appear in the review list
2. Check a song that had a 90%+ confidence match
3. **Expect**: Spotify link was set automatically without admin review

### Cleanup
- Revert any approved/rejected matches if needed

## Journey 11: Song of the Week — Admin Management

**Priority**: P1
**Role**: Admin
**Pages**: `/dashboard` (admin dashboard), Song of the Week management UI
**Existing coverage**: None

### Preconditions
- Logged in as admin
- At least one song exists

### Happy Path

#### Step 1 — Set Song of the Week
1. Navigate to SOTW management (from admin dashboard or dedicated section)
2. **Expect**: Current SOTW shown (or empty state "No Song of the Week set")
3. Search for a song to set as SOTW
4. Select the song
5. Add teacher message: `This week we're focusing on finger-picking technique!`
6. Set active_until date: 7 days from today
7. Click "Set as Song of the Week"
8. **Expect**: Success confirmation
9. **Expect**: The selected song now displayed as current SOTW

#### Step 2 — Verify SOTW visibility on dashboard
1. Navigate to `/dashboard`
2. **Expect**: Song of the Week widget/card visible
3. **Expect**: Song title, author, teacher message all displayed
4. **Expect**: Links (YouTube, Spotify, UG) shown if available on the song

#### Step 3 — Deactivate SOTW
1. Click "Deactivate" / "Remove" on the current SOTW
2. **Expect**: SOTW is deactivated
3. **Expect**: Dashboard no longer shows SOTW (or shows empty state)

#### Step 4 — Set a NEW SOTW (replaces previous)
1. Set a different song as SOTW
2. **Expect**: Previous SOTW is automatically deactivated (DB unique constraint)
3. **Expect**: New SOTW displayed

### Edge Cases

#### E1 — Only one active SOTW at a time
1. Verify in DB: `song_of_the_week` has at most ONE row with `is_active = true`

#### E2 — Teacher message max length (500 chars)
1. Try to set a message with 501+ characters
2. **Expect**: Validation error or truncation

### Cleanup
- Deactivate any test SOTW
