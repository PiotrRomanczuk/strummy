## Severity legend

- **S1** — 5xx on a production user path or data loss
- **S2** — Auth bypass / RLS hole / privilege escalation
- **S3** — Inconsistent contract (shape, status code, schema drift)
- **S4** — Cosmetic / cleanup / dead code

| 8   | Bearer-token user auth is broken           | S2         | Authenticated Bruno run (admin JWT, against local dev server + LAN Supabase, 2026-05-13). `withApiAuth` reads the Bearer token via `authenticateRequest` correctly (resolves user.id), but then calls `loadAuthedProfile` → `createClient()` which is **cookie-only** (`lib/supabase/server.ts:9`). The bearer JWT context is lost, the profile query runs as anon, RLS blocks it, returns null → 403 "Profile not found". Affects ~18 role-checked routes. Verified: admin profile exists in LAN Supabase, direct PostgREST query with same JWT returns the row. Fix: in `loadAuthedProfile` use `createAdminClient()` (service-role, RLS bypass — safe since user.id is already trusted) OR pass the bearer through to a token-aware client. | _open_     |
| 9   | ~111 routes ignore Bearer headers entirely | S2         | Same Bruno run: 111 routes returned 401 even with a valid admin JWT.
