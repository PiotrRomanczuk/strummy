Editorial Lesson Detail — design-mockup fidelity

2026-07-23 · branch feature/lesson-detail-fidelity · route /dashboard/lessons/[id]

What shipped

The editorial Lesson Detail page previously rendered only a hero, a Songs card, and a Notes card. It now matches the design mockup's two-column layout using data that already exists — no new DB columns. New cards and an interactive per-song stepper were added.

Cards / features added

Area Behaviour
Hero Status pill + #<lesson_teacher_number> badge + "with <student>" link (student link kept). Number badge hidden when the lesson has no number.
Lesson info (right col) Scheduled (full date + time), Student (links to /dashboard/users/[studentId] with email sub-line), Teacher name, Sequence ("Lesson #N with <first>").
Assignments / homework Checklist of assignments where assignments.lesson_id = this lesson: tick + strikethrough for completed, due date, link to the assignment. Empty state when none. "+ Add" affordance (→ /dashboard/assignments/new) only for editors.
Continuity Up to 3 previous lessons for the same student (excludes the current one, newest first): #number badge + short date + title/notes summary, each linking to that lesson. Empty state when none.
Per-song progress stepper Interactive 5-stage stepper per song (to_learn → started → remembered → with_author → mastered). Teachers/admins click a segment to advance; writes lesson_songs.status via the existing updateLessonSongStatus(lessonId, songId, status) server action (RLS-scoped, Zod-validated). Students see it read-only.

How to test manually

Prerequisites
  1. cd strummy && npm run dev (points at the dev Supabase stack).
  2. Sign in as the admin/teacher (p.romanczuk@gmail.com) — see CLAUDE.md dev credentials.
  3. Open a lesson that has songs, at least one assignment with lesson_id set, and a student with prior lessons.
Teacher (canEdit) checks — /dashboard/lessons/<id>
  • Hero shows the status pill, a Lesson #N badge, and the student name as a link.
  • Right column shows Lesson info with a full scheduled date/time, a clickable student, teacher name, and the sequence line.
  • Assignments card lists lesson homework with due dates; completed items show a tick + strikethrough; the "+ Add" link is visible.
  • Continuity card lists up to 3 earlier lessons with that student; each row navigates to that lesson.
  • In the Songs card, click a stage segment on a song — the fill advances and the label under it updates. Reload: the new stage persists (written to lesson_songs.status).
Student (read-only) checks
  • Sign in as the demo student (emma@strummy.app) and open one of their lessons.
  • The song steppers render as static bars — no clickable segments, no "+ Add" on Assignments, no "Edit lesson" link.
Edge cases
  • Lesson with no songs → "No songs attached to this lesson yet."
  • Lesson with no homework → "No homework attached to this lesson."
  • Student's first lesson (no prior) → "No previous lessons with <first>."
  • Lesson missing a number → hero badge hidden, sequence reads "With <first>".
  • A song with a null/unknown status → stepper renders unfilled with a "Not started" label.
