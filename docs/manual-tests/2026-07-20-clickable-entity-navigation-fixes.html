P0 · bug  Student Health “View Profile” route

What changed: the desktop “View Profile” icon and the mobile card both linked to /dashboard/users?id=X — a query param on the roster list, which dead-ends there. Both now link to the detail route /dashboard/users/[id]. The mobile student name is now a link too (it was plain text; the desktop table already linked it). components/dashboard/health/StudentHealthTable.tsx

  1. Log in as admin/teacher → go to /dashboard/health.
  2. Desktop: click the person icon in the Actions column of any row. → lands on that student's profile /dashboard/users/<id>, not the roster.
  3. Also click the student's name cell → same profile.
  4. Mobile (narrow the window < 768px): tap the person icon and the student name at the top of the card. → both open the student profile.

P1  Notifications become clickable & deep-link to the record

What changed (two parts):

  • Inbox now navigates. The full-page inbox rendered every row as plain text and never used action_url. The title/body is now a link when the notification has an action URL, and clicking also marks it read. (The “Mark read” button stays separate — no nested links.) components/notifications/editorial/NotificationsEditorial.List.tsx
  • Deep links, not list pages. New notifications now derive action_url from the notification's own entity reference (entity_type + entity_id), so a lesson notification opens /dashboard/lessons/<id> instead of the generic /dashboard/lessons list. Covers lesson, song, assignment, and student. lib/services/in-app-notification-service.ts · lib/services/notification-in-app-content.ts (entityDetailUrl)

Test the inbox link:

  1. Go to /dashboard/notifications. Any notification with an action opens its target on click; the unread highlight clears.

Test the deep-link derivation (needs a freshly-created notification — existing rows keep their old URL):

  1. Trigger a lesson notification — e.g. open a completed lesson and send its recap (app/dashboard/lessons/actions.ts), or run the lesson-reminders cron locally.
  2. Open the new notification from the bell or the inbox. → it opens the specific lesson /dashboard/lessons/<id>, not the lessons list.
  3. Quick DB check (new rows): action_url should be /dashboard/lessons/<uuid> for a lesson notification.

P1  Song titles link in Repertoire & Practice history

What changed: song titles were plain text in the repertoire grid and the practice-history list — despite the near-identical student-detail repertoire already linking. Both now link to /dashboard/songs/[id]. Practice sessions with no song (“General technique”) stay plain text. components/repertoire/editorial/RepertoireCard.tsx · components/practice/editorial/PracticeHistoryList.tsx

  1. Go to /dashboard/repertoire → click a song title on any card. → opens that song's detail page.
  2. Go to /dashboard/practice → click a song title in the history list. → opens that song. “General technique” entries remain non-clickable.
