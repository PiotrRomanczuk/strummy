Hide the AI UI from the frontend for now

Date: 2026-07-20  ·  Author: Claude  ·  Branch: feature/lessons-display

What was done

The app is slimmed down to the core teaching loop — Lessons, Songs, Assignments (plus Students) — and every AI-powered surface is hidden from the UI. Nothing was deleted: routes, server actions, and the AI components all remain in the codebase. Everything is gated behind one flag so it can be brought back in a single edit.

The switch: lib/config/features.ts → export const SHOW_AI_FEATURES = false;
Flip it to true to restore the AI nav items and the in-form generators everywhere, at once.

Sidebar (desktop + mobile)

All navigation goes through getMenuGroups() in components/navigation/menuConfig.ts, so these changes apply to the desktop shell, the mobile "More" menu, and the sidebar helpers identically.

Item Before Now How it's hidden
Lessons / Songs / Assignments visible visible —
Students visible visible —
AI Assistant visible hidden gated on SHOW_AI_FEATURES
AI Chat visible hidden gated on SHOW_AI_FEATURES
Calendar visible hidden added to CORE_LOOP_HIDDEN_ITEMS
Fretboard visible hidden added to CORE_LOOP_HIDDEN_ITEMS

In-form AI generators

The "Generate with AI" affordances embedded in the editorial forms are hidden. The underlying note/description fields stay fully editable — only the AI button disappears.

Location Hidden component Core field kept?
New/Edit Lesson form LessonNotesAI ("Generate Lesson Notes") Yes — notes field unaffected
Lesson detail (edit view) PostLessonSummaryAI Yes — read-only summary card only
New/Edit Song form SongNotesAI ("Generate Song Notes") Yes — textarea[name="notes"] still renders
Create Assignment form AssignmentAI ("Generate Assignment") Yes — description textarea unaffected

The song-form notes placeholder is also swapped from "…or generate them with AI above." to a neutral "Teaching tips and practice suggestions." while AI is off.

How to test manually

Setup
  1. cd strummy && npm run dev
  2. Sign in as the admin/teacher: p.romanczuk@gmail.com / test123_admin

1. Sidebar shows only the core loop

  1. Open /dashboard. In the sidebar, confirm you see Lessons, Songs, Assignments, Students and Home.
  2. Confirm there is no AI Assistant, AI Chat, Calendar, or Fretboard entry.
  3. Shrink the window to mobile width → open the "More" menu → confirm the same items are absent there too.

2. In-form AI buttons are gone (but fields work)

  1. Song: go to /dashboard/songs → New Song. Under "Teaching notes" there should be no "Generate Song Notes" button — just the notes textarea, which you can type into and save.
  2. Lesson: go to /dashboard/lessons → New Lesson. Confirm there is no "Generate Lesson Notes" button. Open an existing lesson's detail/edit view → confirm no post-lesson AI summary card.
  3. Assignment: go to /dashboard/assignments → create one. Confirm there is no "Generate Assignment" button; the description textarea still works.

3. Routes still exist (hide is nav-only)

  1. Manually visit /dashboard/ai, /dashboard/ai/chat, /dashboard/calendar, /dashboard/fretboard by typing the URL. They should still load — hiding is visibility-only, not removal.

4. Re-enable check (optional)

  1. Set SHOW_AI_FEATURES = true in lib/config/features.ts, save, and refresh.
  2. The AI Assistant / AI Chat sidebar items and all four in-form "Generate…" buttons should reappear. (Calendar and Fretboard stay hidden — they are parked separately, not tied to the AI flag.)
  3. Revert the flag back to false.
