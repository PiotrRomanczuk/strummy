Assignment: daily practice target + submission-type selector

2026-07-23 · branch feature/assignment-target-submission

What was built

Two Assignment-Form gaps from the design mockup are now wired end to end (migration → types → Zod schema → form → detail display):

  • Daily practice target — an optional select on the assignment form offering 5 10 15 20 min/day (plus No target). Stored in assignments.daily_target_minutes (nullable integer, > 0).
  • Submission type selector — a segmented toggle declaring the expected proof: Self-report / Audio recording / Video / Note. Stored in assignments.submission_type (text, default self_report, CHECK-constrained to the four values).
Scope note: this is the submission-type selector only — it declares what proof is expected. Actual audio/video upload is intentionally deferred to a later wave.

Where it shows up

  • Create form (/dashboard/assignments/new): "Daily target" select next to the due date; a "How should they submit?" segmented toggle in the PROOF section (defaults to Self-report).
  • Edit form (/dashboard/assignments/<id>/edit): both fields are seeded from the saved assignment.
  • Detail page (/dashboard/assignments/<id>): the Brief card shows Target · 10 min/day (omitted when there is no target) and always Submit as · <label>.

How to manually test

Prerequisites

  1. Apply the migration to the dev DB (uwh StudentDevelopment stack): 
    docker exec -i supabase_db_StudentDevelopment psql -U postgres -d postgres \
  < supabase/migrations/20260723120100_assignments_target_submission.sql
  2. Run the app: npm run dev. Sign in as a teacher/admin (see CLAUDE.md dev creds).

Create flow

  1. Go to /dashboard/assignments/new. Pick a student, enter a title.
  2. Set Daily target to 15 min/day. Click the Video submission pill (it turns dark/selected).
  3. Click Create assignment. Expected: redirect to the new assignment detail.
  4. On the detail page, the Brief card shows Target · 15 min/day and Submit as · Video.
  5. Edge case: leave Daily target on No target and don't touch the toggle → detail shows no Target line and Submit as · Self-report.

Edit flow

  1. Open /dashboard/assignments/<id>/edit for the assignment above.
  2. Confirm the Daily target select reads 15 min/day and the Video pill is pre-selected.
  3. Change target to 5 min/day, pick Audio recording, save. Detail reflects the new values.

DB / constraint checks (optional)

  • A row with submission_type = 'podcast' is rejected by assignments_submission_type_check.
  • A row with daily_target_minutes = 0 is rejected by assignments_daily_target_minutes_positive; NULL is allowed.
