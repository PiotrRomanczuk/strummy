Lesson duration + in-person/video format

Date: 2026-07-23 · Branch: feature/lesson-duration-format · schema → form → display

What was built

The two Lesson-Form gaps from the design mockup are now wired end to end: a Duration select (30 / 45 / 60 min) and a Format toggle (In-person / Video call). Both persist on lesson create and edit, seed correctly when editing an existing lesson, and now surface on the read surfaces.

  • Migration adds two nullable columns to public.lessons: duration_minutes integer and format text, each guarded by an idempotent CHECK constraint (format in ('in_person','video'), duration_minutes between 15 and 180). No backfill — existing rows stay NULL and the form supplies defaults (45 min / in_person) on the next save. RLS inherits the table's existing policies.
  • Form: Duration select + Format segmented toggle live in Section I of the editorial lesson form. Duration defaults to 45; Format defaults to In-person.
  • Preview panel shows … · 45 min next to the scheduled time.
  • Lesson detail meta line reads e.g. Lesson · Mon, Jul 20, 2026, 3:00 PM · 45 min · In person (duration/format omitted cleanly when NULL).
  • Lessons list Time column appends the duration: Mon · 4:00 PM · 45 min.

How to test manually

Prerequisites: apply the migration to the dev Supabase stack, then run the app.

  1. Apply migration on uwh: docker exec -i supabase_db_StudentDevelopment psql -U postgres -d postgres < supabase/migrations/20260723120000_lessons_duration_format.sql, then regenerate types with supabase gen types (see note below).
  2. npm run dev and sign in as the admin/teacher (see CLAUDE.md dev creds).
  3. Go to /dashboard/lessons/new. Confirm the Duration select shows 30/45/60 (defaulting to 45) and the Format toggle shows In-person (selected) / Video call. The preview card should read … · 45 min.
  4. Pick a student + time, change Duration to 60, click Video call, and create. On the new lesson detail page the meta line should read … · 60 min · Video call.
  5. Return to /dashboard/lessons. The new row's Time column should show · 60 min.
  6. Open the lesson's Edit page. Duration should be pre-selected to 60 and Video call pre-pressed. Change them back to 45 / In-person, save, and confirm the detail + list update.
  7. Edge — legacy rows: a lesson that predates the migration (duration/format NULL) should render its detail meta line and list row with no duration/format fragment, and its edit form should fall back to 45 / In-person.
