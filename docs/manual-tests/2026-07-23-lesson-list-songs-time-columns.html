Lessons list — Songs & Time columns + sort/year filters

Date: 2026-07-23 · Branch: feature/lesson-list-columns · Area: /dashboard/lessons (editorial)

What was built

The editorial Lessons list now matches its mockup, using only existing data (no new DB columns):

  • Songs column — per-lesson count of attached lesson_songs plus up to four colored progress dots (dot colour by each song's lesson_songs.status).
  • Time column — the time-of-day of scheduled_at (reuses the existing formatLessonClock formatter). No duration is shown — there is no duration column in the schema.
  • #number badge — the lesson's lesson_teacher_number rendered in the title cell.
  • Sort toggle — a ?sort= URL-param link (Newest/Oldest first) in the filter row; activating it flips the grouped timeline into a flat, fully-sorted table (mockup behaviour).
  • Year filter — ?year= URL-param chips (All + current year and the two prior); narrows the query by the calendar year of scheduled_at.

Status chips, sort, and year are all plain <Link>s (Server-Component friendly, no client JS) and preserve each other in the query string. With no sort= param the list keeps the existing date-grouping (Today / This week / Upcoming / Past). Mobile keeps the existing responsive stack (grid-cols-1 under the md breakpoint).

How to test manually

  1. Run npm run dev and sign in (teacher/admin: full columns; student: no student/teacher columns).
  2. Open /dashboard/lessons. Confirm each row shows a #number badge, a Songs count with colored dots, and a Time value. The list is grouped by time bucket.
  3. Click Newest first in the filter row. URL gains ?sort=oldest; the section headers disappear and rows render as one flat, oldest-first table. Click again → ?sort=newest, newest-first flat table. The summary line reads "…· sorted by newest/oldest first".
  4. Click a Year chip (e.g. 2025). URL gains ?year=2025; only that year's lessons show. Click All to clear it. Combine with a status chip and the sort toggle — all three params persist together.
  5. Open a lesson with several songs and confirm the dot colours match song status (to_learn = muted, started = blue, remembered = gold, with_author = violet, mastered = green).
  6. Edge cases: a lesson with 0 songs shows "0 songs" and no dots; 1 song shows the singular "song"; more than 4 songs shows only the first four dots.
