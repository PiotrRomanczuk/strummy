Date: 2026-07-20 · PR: C of the assignments teacher-POV slice · Branch: feature/assignments-triage

What was built

The assignment_templates table had tested CRUD actions and a schema but no table in the current migration set and no UI. This PR gives teachers a reusable template library:

  • A migration (re)creating assignment_templates using this stack's RLS helpers, with a new checklist column so a template can carry sub-tasks.
  • Real editorial routes: /dashboard/assignments/templates (list), /templates/new, and /templates/[id] (edit + delete behind a confirm dialog).
  • A “Start from template” picker on the New-assignment form that prefills title, brief, and checklist.
  • A teacher-only “Templates” link on the assignments list header.

How to manually test (after applying the migration)

Credentials: teacher p.romanczuk@gmail.com / test123_admin.

  1. Go to /dashboard/assignments → click Templates in the header. The empty state invites you to create one.
  2. Click + New template. Give it a title, a brief, and 2–3 checklist items. Save → it appears in the list (with an “N checklist items” badge).
  3. Open the template → edit the title → save → the change persists.
  4. Go to + New assignment. Use “Start from template” → pick the template → title, brief, and checklist prefill. Choose a student and save.
  5. Back in Templates, open a template → Delete → confirm in the dialog → it disappears. Assignments already created from it are unaffected.
  6. Sign in as a student and confirm there is no Templates link and no /dashboard/assignments/templates access (redirects to the assignments list).
