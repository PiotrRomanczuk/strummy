Song Cover-Image Upload

Date: 2026-07-23 · Branch: feature/song-cover-upload

What was built

Teachers/admins can now attach a cover image to a song directly from the editorial create and edit forms — either by uploading a file (stored in a new public song-covers Supabase Storage bucket) or by pasting a URL manually. The resolved public URL is submitted through the existing native form as cover_image_url and persisted to the songs row.

  • New migration 20260723000001_song_covers_storage_bucket.sql — public song-covers bucket (2 MB limit, PNG/JPEG/WebP/GIF), public read, and insert/update/delete gated to public.is_admin() OR public.is_teacher() (songs are catalog entities, so writes are role-gated, not owner-folder-gated like avatars).
  • Helper lib/storage/songCover.ts — validateSongCoverFile (pure) and uploadSongCover. Create-flow objects use a random UUID name; edit-flow objects use a deterministic <songId>/cover.<ext> path with upsert.
  • Client component components/songs/editorial/form/SongFormEditorial.CoverUpload.tsx — file input + manual URL fallback + Remove button, error surfaced via data-testid="song-cover-upload-error".
  • Both server actions (createSongAction, updateSongAction) now read and persist cover_image_url.
Storage-less stacks: the upload button is hidden when NEXT_PUBLIC_SONG_COVER_UPLOAD_ENABLED=false. The StudentDevelopment dev stack has no storage-api container, so set the flag there — the manual URL text field still works, so the feature degrades gracefully to paste-a-URL.

How to test manually

  1. Apply the migration to a stack that has Storage running (StudentProduction / Cloud): 
    docker exec -i supabase_db_<stack> psql -U postgres -d postgres \
  < supabase/migrations/20260723000001_song_covers_storage_bucket.sql
  2. Run the app: npm run dev, sign in as an admin or teacher.
  3. Create flow: go to /dashboard/songs/new. In section I · IDENTITY, find the Cover image field. Click Upload image, choose a PNG/JPEG/WebP/GIF ≤ 2 MB. The URL field fills with the public URL. Submit and confirm the saved song has the cover.
  4. Edit flow: open an existing song's /edit page. The Cover image field is pre-filled from the stored value. Upload a replacement (overwrites <songId>/cover.<ext>) or click Remove to clear it, then Save.
  5. Manual URL fallback: paste any https://… URL into the field and Save.

Edge cases to check

Input Expected
Non-image file (e.g. .pdf/.txt) Inline error "Please choose a PNG, JPEG, WebP, or GIF image."; no upload attempted.
Image > 2 MB Inline error "Image must be 2 MB or smaller."
Storage service unreachable Inline error with the storage message (e.g. "Bucket not found"); form still submittable via manual URL.
Remove on an existing cover Field clears; saving persists cover_image_url = null.
Non-teacher/non-admin Edit page redirects away; RLS also rejects storage writes.
