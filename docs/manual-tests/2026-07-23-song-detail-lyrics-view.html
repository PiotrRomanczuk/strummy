2026-07-23 · fix/song-detail-lyrics-view

What was done

The lyrics_with_chords column was written by both song forms (create & edit) but read by no screen — teachers could type lyrics that appeared nowhere. This fix surfaces them on the song detail page.

  • New component SongLyricsCardEditorial renders the stored text verbatim in a monospace block so the teacher's hand-aligned "chords over lyrics" layout is preserved (no re-parsing).
  • [Section] header lines (e.g. [Verse 1], [Chorus]) are lifted into gold uppercase labels with the brackets stripped.
  • Mounted in the Overview column of SongDetailEditorial, directly below the Chords card — so both teachers and students see it.
  • Renders nothing when a song has no lyrics, so lyric-less songs show no empty card.
  • Long lines scroll horizontally inside the card (alignment never wraps/breaks).

How to test it for real

  1. Start the dev server: npm run dev (local Supabase per the E2E runbook if needed).
  2. Edit any song and paste chords-over-lyrics text into the Lyrics with chords field, e.g.:
    [Verse 1]⏎C        G⏎Today is gonna be the day…. Save.
  3. Open that song's detail page /dashboard/songs/<id>. The Lyrics card appears under Chords, in the Overview column.
  4. Confirm: chord alignment is preserved; [Section] headers show as gold labels; a very long line scrolls horizontally rather than wrapping.
  5. Open a song without lyrics → no Lyrics card renders.
  6. Log in as a student and open a song with lyrics → the Lyrics card is visible to them too.
