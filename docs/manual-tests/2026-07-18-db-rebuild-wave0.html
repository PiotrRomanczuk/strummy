What was built

The first two migrations of the from-scratch database rebuild. The old supabase/migrations/ (three incompatible generations, never runnable on a fresh DB) was moved to supabase/migrations_archive/; the chain now starts clean.

File What it creates
20260718090000_foundation.sql Extensions citext, pgcrypto. Closed-set enums difficulty_level, lesson_status, assignment_status (5 values — legacy pending dropped), lesson_song_status (unified, includes slow_tempo from day one so app/DB enums never drift). Single set_updated_at() trigger fn.
20260718090100_profiles.sql profiles table — independent id PK + nullable user_id FK to auth.users (null = shadow, added later); citext email; role boolean flags. Role helpers is_admin/is_teacher/is_student/is_admin_or_teacher (SECURITY DEFINER, search_path pinned, read via (select auth.uid())). One clean handle_new_user() signup trigger on auth.users. Explicit grants to authenticated/service_role. RLS: self read/update, admin all.

Resulting behavior

  • Signing up a new auth user auto-creates a matching profiles row with names pulled from user metadata (full name derived from first/last when not supplied).
  • A signed-in user sees and edits only their own profile; an admin sees all; a student cannot modify another user's row — enforced by RLS, not app code.
  • No user_roles table — the profile boolean flags are the single source of truth (matches what the app already reads).
